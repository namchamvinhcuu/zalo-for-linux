# Architecture

This document explains how Zalo for Linux works under the hood.

## Overview

This project is not a from-scratch rewrite of Zalo. Instead, it:

1. Downloads the official macOS `.dmg` file.
2. Uses `7z` to extract the `app.asar` archive, which contains the main application logic written in JavaScript.
3. Removes incompatible native macOS files.
4. Wraps the extracted application in a minimal, Linux-compatible Electron shell.
5. Uses `electron-builder` to package everything into a single, portable `AppImage` file.

## Directory Structure

│   ├── download-dmg.js      # Download macOS DMG

│   ├── prepare-zadark.js    # Build ZaDark dark-mode plugin
│   ├── build.js             # Package into AppImage

│       ├── patch-titlebar.js
│       ├── patch-sqlite3.js
│       ├── patch-db-cross-v4.js
│       ├── patch-clipboard-paste.js
│       ├── patch-tray-icon.js
│       ├── patch-tray-badge.js
│       └── patch-zcall.js
├── plugins/
│   ├── zadark/              # Dark mode extension (git submodule: quaric/zadark)
│   ├── zadark-css/          # Runtime CSS/font injection for ZaDark
│   ├── screenshot/          # Native Linux screenshot (intercepts screen-capture IPC)
│   ├── tray-badge/          # Unread indicator: tray red dot + dock count (D-Bus)

## Build Pipeline

`npm run main` runs the full pipeline:

1. **check-versions** — Compares local vs latest Zalo release
2. **download-dmg** — Downloads `ZaloSetup-universal-<version>.dmg` to `temp/`
3. **prepare-zadark** — Builds ZaDark assets (cloned from submodule)
4. **prepare-app** — Extracts `app.asar` to `app/` and applies patches
   via `scripts/patches/`:
   - `patch-titlebar.js` — Enable native title bar
   - `patch-sqlite3.js` — Replace macOS sqlite3 with Linux binary
   - `patch-db-cross-v4.js` — Build db-cross-v4 from source + patch binding.js + fix Linux platform ID
   - `patch-clipboard-paste.js` — Enable pasting clipboard images into chats
   - `patch-tray-icon.js` — Fix Zalo's broken native tray icon on Linux
   - `patch-tray-badge.js` — Un-gate the unread badge so the tray dot / dock count work
   - `patch-zcall.js` — Un-gate call buttons + wire the engine spawn to the route-B Wine bridge
5. **build** — Packages everything into `dist/Zalo-<version>.AppImage`

## Extracted App Patches

When the macOS `app.asar` is extracted, certain modifications are needed
for Linux compatibility. These are applied in `scripts/prepare-app.js`:

| Patch | Why |
|-------|-----|
| `patch-titlebar.js` | `T,frame:!1` → `T,frame:!0` — Enable native title bar |
| `patch-sqlite3.js` | macOS `node_sqlite3.node` → Linux one — Avoid "invalid ELF header" crash |
| `patch-db-cross-v4.js` | Build db-cross-v4 from source, patch `binding.js` for Linux, fix `case"LINUX":return 25;` → `24` |
| `patch-clipboard-paste.js` | Add clipboard native helpers + a paste→drag-drop interceptor so images paste into chats (issue #23) |
| `patch-tray-icon.js` | Zalo's tray loads `favicon.ico`, which `nativeImage` renders broken on Linux → point it at `favicon-512x512.png` |
| `patch-tray-badge.js` | Un-gate the unread badge (hidden on non-macOS) so the tray red dot and dock count render on Linux |
| `patch-zcall.js` | Un-gate the voice-call buttons (Darwin version gate) and inject a `linux` branch into the engine-spawn picker `W()` so calls run through the route-B Wine bridge instead of the missing macOS `ZaloCall` binary |

For more details on the native addons, see [nativelibs/README.md](./nativelibs/README.md).

## Voice calls (route B — Wine bridge)

Zalo's call engine ships only as a platform-native binary (`ZaloCall`), absent
from the Linux build. Rather than reimplement it, the AppImage **bundles a
self-contained Wine plus the Windows call engine** and bridges to it:

- `patch-zcall.js` stages `app/zcall-bridge/` (portable Wine, the trimmed engine,
  `bridge-daemon.sh`/`.js`, and the PE32 `bridge-shim.exe`) and wires `W()` to
  spawn the bridge daemon where the native engine would be.
- At call time, Electron talks the same unix-socket control protocol it always
  did; `bridge-daemon.js` translates it to the Windows engine running under Wine
  (reassembling chunked frames, synthesizing acks) and back.
- The bundled Wine runs in a dedicated prefix that the daemon initializes on
  first use, so no system Wine or manual setup is needed. The launcher is
  node-independent (system `node` or the bundled Electron run as Node).

Audio 1:1 calls work; video is not supported yet. Source for the bridge lives in
`nativelibs/zcall-bridge/`.

## Why an Electron shell?

The extracted Zalo app is an Electron app designed for macOS. We:

1. Provide a minimal `main.js` that:
   - Loads the runtime plugins (ZaDark CSS injection, screenshot, tray-badge)
   - Loads `app/main-dist/main.js` from the extracted app
2. Use our own `package.json` with Linux-specific electron-builder config
3. Rename the extracted `package.json` to `package.json.bak` to avoid conflicts

## What lives in the extracted app/?

Everything in `app/` is **regenerated** by the build pipeline and is
gitignored. It contains:

- `main-dist/main.js` — Zalo's main process bundle
- `main-dist/preload-*.js` — Preload scripts
- `main-dist/compact-app.js` — Utility process
- `pc-dist/` — Renderer/UI bundles
- `native/nativelibs/` — Native addons (some replaced with Linux versions)

The extracted code is proprietary to VNG/Zalo. We don't modify it directly
in version control — patches are applied each build via `prepare-app.js`.
